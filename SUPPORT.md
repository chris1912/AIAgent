# Support / 支持说明

## Supported Requests / 支持范围

GitHub Issues may be used for reproducible defects, installation problems, documentation corrections, and focused feature proposals concerning code in this repository.

GitHub Issues 可用于反馈本仓库代码的可复现缺陷、安装问题、文档修正和聚焦的功能建议。

Before opening an issue, read the selected project's README and run its offline tests when possible. Include sanitized version and error information.

提交 Issue 前，请阅读对应项目 README，并尽可能运行离线测试，同时提供已脱敏的版本与错误信息。

## Out of Scope / 不支持范围

- Provider account recovery, billing, quota increases, or VPN configuration. / 提供方账户恢复、计费、额度提升或 VPN 配置。
- Guaranteed availability or performance of external models and CLIs. / 保证外部模型和 CLI 的可用性或性能。
- Review of private prompts, credentials, full provider logs, or proprietary repositories posted publicly. / 公开审查私人提示词、凭据、完整提供方日志或专有仓库。
- Emergency or production incident response. / 紧急事件或生产事故响应。

Security vulnerabilities must follow [`SECURITY.md`](SECURITY.md) instead of public support channels.

安全漏洞必须按 [`SECURITY.md`](SECURITY.md) 报告，不得使用公开支持渠道。
